# コスト比較: DXECO vs 自社開発POC

## 前提条件

| 項目 | 値 |
|------|-----|
| 利用人数 | 約500名 |
| 管理対象SaaS | 約260件 |
| DXECOプラン | 500名・オプション込み（監査ログ+IP制限） |
| 自社インフラ | Azure Container Apps + PostgreSQL |

---

## 1. 年間コスト比較

### DXECO

| 項目 | 年額 |
|------|------|
| 基本料金（500名） | 180万円 |
| 監査ログオプション | 12万円 |
| IP制限オプション | 12万円 |
| **合計** | **204万円** |

### 自社開発

| 項目 | 年額 |
|------|------|
| Azure インフラ | 約6万円 |
| 保守工数（セキュリティパッチ等） | 年数日程度 |
| **合計** | **約6万円 + 保守工数** |

---

## 2. 5年間累計コスト

| 期間 | DXECO | 自社開発 | 差額 |
|------|-------|----------|------|
| 1年目 | 204万円 | 約6万円 + POC開発2週間 | 約198万円 |
| 3年累計 | 612万円 | 約18万円 | 約594万円 |
| 5年累計 | **1,020万円** | **約30万円** | **約990万円** |

※ 自社開発の保守工数は、テスト・CI整備済みのため年数日程度の想定

## 4. DXECOのコスト構造の問題点

- セキュリティ機能（監査ログ・IP制限）が有料オプション → 個人情報を扱う以上は事実上必須
- 100名刻み・途中減額不可 → 段階的導入との相性が悪い
- カスタム連携は有償開発 → 不動産業界特化サービスとの連携は追加費用
- 価格は資料非掲載（要問い合わせ）→ 透明性が低い

## 5. 自社開発の優位点

- **5年間で約990万円のコスト削減**
- 現場が最も求める「申請・承認フロー」を標準装備（DXECOにはない）
- Entra ID連携で退職者を自動検出（DXECOはCSV手動突合せ）
- 監査ログ・IP制限が追加費用ゼロ
- 要望に応じて自由にカスタマイズ可能
- テスト・CI整備済みで保守コストが低い
