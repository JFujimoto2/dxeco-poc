# SaaSセキュリティ管理

## 概要
SaaS台帳にセキュリティ属性を追加し、ダッシュボードでセキュリティリスクを可視化する。部署別のリスク把握を可能にする。

## セキュリティ属性

SaaS台帳に以下3つの属性を追加。

| 属性 | カラム | 値 |
|------|--------|-----|
| 個人情報取扱い | `handles_personal_data` | boolean |
| 認証方式 | `auth_method` | SSO / パスワード / MFA / その他 |
| データ保存先 | `data_location` | 国内 / 海外 / 不明 |

## ダッシュボード セキュリティリスクカード

| カード | 内容 |
|--------|------|
| SSO未対応リスク | 個人情報を扱うSaaSのうちSSO未対応の件数 |
| 海外データ保存リスク | 個人情報を扱うSaaSで海外にデータ保存している件数 |
| 部署別リスク | 部署ごとのリスクSaaS利用状況 |

## フィルタリング

### SaaS台帳（`/saases`）
- 部署、認証方式、データ保存先でフィルタ可能

### アカウント管理（`/saas_accounts`）
- 部署フィルタを追加

## CSV入出力
CSVインポート・エクスポートに3つのセキュリティ属性カラムを追加。

## アクセス権限
セキュリティ属性の編集はadmin・managerのみ。ダッシュボードのリスクカードは全ロールに表示。
