## 1. Azure CLI セットアップ

GUI のみで構築する場合もステップ9（初回デプロイ）で Docker ビルド & プッシュに CLI が必要。

### 1-1. Azure CLI インストール

```bash
# Linux (WSL含む)
curl -sL https://aka.ms/InstallAzureCLIDeb | sudo bash

# バージョン確認
az version
```

### 1-2. ログイン

```bash
az login

# サブスクリプションを確認・選択
az account list -o table
az account set --subscription "<サブスクリプション名 or ID>"
```

### 1-3. 必要な拡張機能

```bash
az extension add --name containerapp --upgrade
```

## 4. PostgreSQL Flexible Server 作成

### CLI

```bash
# サーバー作成
az postgres flexible-server create \
  --name psql-dxceco-poc \
  --resource-group rg-dxceco-poc \
  --location japaneast \
  --tier Burstable \
  --sku-name Standard_B1ms \
  --storage-size 32 \
  --version 16 \
  --admin-user dxcecopoc \
  --admin-password '<強力なパスワード>' \
  --yes

# データベース4個を作成
for DB in dxceco_poc_production dxceco_poc_production_cache dxceco_poc_production_queue dxceco_poc_production_cable; do
  az postgres flexible-server db create \
    --server-name psql-dxceco-poc \
    --resource-group rg-dxceco-poc \
    --database-name "$DB"
done

# ファイアウォール: Azureサービスからのアクセスを許可
az postgres flexible-server firewall-rule create \
  --name psql-dxceco-poc \
  --resource-group rg-dxceco-poc \
  --rule-name AllowAzureServices \
  --start-ip-address 0.0.0.0 \
  --end-ip-address 0.0.0.0
```

### GUI

1. Azure Portal → **Azure Database for PostgreSQL フレキシブル サーバー** → **作成**
2. **基本** タブ:
   - **サブスクリプション**: 会社のサブスクリプション
   - **リソース グループ**: `rg-dxceco-poc`
   - **サーバー名**: `psql-dxceco-poc`（グローバルで一意）
   - **リージョン**: `Japan East`
   - **PostgreSQL のバージョン**: `16`
   - **ワークロードの種類**: `開発` を選択（B1ms が選ばれる）
   - **コンピューティングとストレージ**: `Burstable B1ms`（1 vCore / 2 GiB RAM）/ ストレージ `32 GB`
   - **管理者ユーザー名**: `dxcecopoc`
   - **パスワード**: 強力なパスワードを設定
3. **ネットワーク** タブ:
   - **接続方法**: `パブリック アクセス`
   - **ファイアウォール規則**: 「Azure 内の任意の Azure サービスにこのサーバーへのパブリック アクセスを許可する」に **チェック**
   - （外部からDB接続する場合）「現在のクライアント IP アドレスを追加する」に **チェック**
4. **確認および作成** → **作成**（3〜5分かかる）
5. 作成完了後、**データベース**（左メニュー）→ **追加** で以下4つを作成:
   - `dxceco_poc_production`
   - `dxceco_poc_production_cache`
   - `dxceco_poc_production_queue`
   - `dxceco_poc_production_cable`

> **メモ:** 「概要」ページの **サーバー名**（`psql-dxceco-poc.postgres.database.azure.com`）を控える。

## 5. Container Apps 環境・アプリ作成

### CLI

```bash
# Container Apps 環境を作成
az containerapp env create \
  --name cae-dxceco-poc \
  --resource-group rg-dxceco-poc \
  --location japaneast

# ACR 認証情報を取得
ACR_LOGIN_SERVER=$(az acr show --name acrdxcecopoc --query loginServer -o tsv)
ACR_PASSWORD=$(az acr credential show --name acrdxcecopoc --query "passwords[0].value" -o tsv)

# DATABASE_URL を組み立て
DATABASE_URL="postgres://dxcecopoc:<パスワード>@psql-dxceco-poc.postgres.database.azure.com:5432/dxceco_poc_production?sslmode=require"

# Container App を作成
az containerapp create \
  --name app-dxceco-poc \
  --resource-group rg-dxceco-poc \
  --environment cae-dxceco-poc \
  --image "${ACR_LOGIN_SERVER}/dxceco-poc:latest" \
  --registry-server "$ACR_LOGIN_SERVER" \
  --registry-username acrdxcecopoc \
  --registry-password "$ACR_PASSWORD" \
  --target-port 3000 \
  --ingress external \
  --cpu 0.5 \
  --memory 1Gi \
  --min-replicas 0 \
  --max-replicas 1 \
  --env-vars \
    "RAILS_ENV=production" \
    "RAILS_MASTER_KEY=<config/master.keyの値>" \
    "DATABASE_URL=${DATABASE_URL}" \
    "DXCECO_POC_DATABASE_PASSWORD=<DBパスワード>" \
    "DB_HOST=psql-dxceco-poc.postgres.database.azure.com" \
    "DB_USERNAME=dxcecopoc" \
    "APP_URL=https://<FQDN>" \
    "SOLID_QUEUE_IN_PUMA=true"
```

### GUI

#### 5-A. Container Apps 環境の作成

1. Azure Portal → **Container Apps 環境** → **作成**
2. **基本** タブ:
   - **サブスクリプション**: 会社のサブスクリプション
   - **リソース グループ**: `rg-dxceco-poc`
   - **環境名**: `cae-dxceco-poc`
   - **リージョン**: `Japan East`
   - **環境の種類**: `消費のみ`
3. **監視** タブ:
   - **Azure Log Analytics ワークスペース**: 新規作成（デフォルトでOK）
4. **確認および作成** → **作成**

#### 5-B. Container App の作成

1. Azure Portal → **コンテナー アプリ** → **作成**
2. **基本** タブ:
   - **サブスクリプション**: 会社のサブスクリプション
   - **リソース グループ**: `rg-dxceco-poc`
   - **コンテナー アプリ名**: `app-dxceco-poc`
   - **Container Apps 環境**: `cae-dxceco-poc`（ステップ5-Aで作成したもの）
3. **コンテナー** タブ:
   - **イメージのソース**: `Azure Container Registry`
   - **レジストリ**: `acrdxcecopoc`
   - **イメージ**: `dxceco-poc`
   - **イメージ タグ**: `latest`
   - **CPU とメモリ**: `0.5 vCPU, 1 Gi`
   - **環境変数** を以下の通り追加:

| 名前 | ソース | 値 |
|------|--------|-----|
| `RAILS_ENV` | 手動エントリ | `production` |
| `RAILS_MASTER_KEY` | 手動エントリ | `<config/master.key の値>` |
| `DATABASE_URL` | 手動エントリ | `postgres://dxcecopoc:<パスワード>@psql-dxceco-poc.postgres.database.azure.com:5432/dxceco_poc_production?sslmode=require` |
| `DXCECO_POC_DATABASE_PASSWORD` | 手動エントリ | `<DBパスワード>` |
| `DB_HOST` | 手動エントリ | `psql-dxceco-poc.postgres.database.azure.com` |
| `DB_USERNAME` | 手動エントリ | `dxcecopoc` |
| `APP_URL` | 手動エントリ | `https://<FQDN>`（作成後に更新） |
| `SOLID_QUEUE_IN_PUMA` | 手動エントリ | `true` |

4. **イングレス** タブ:
   - **イングレス**: `有効`
   - **イングレス トラフィック**: `どこからでもトラフィックを受け入れる`
   - **イングレスの種類**: `HTTP`
   - **ターゲット ポート**: `3000`
5. **確認および作成** → **作成**
6. 作成後、**概要** ページの **アプリケーション URL** をメモし、`APP_URL` 環境変数を更新

## 7. Entra ID アプリ登録

> このステップは Azure Portal（GUI）での操作が基本。

### 7-1. アプリの登録

1. Azure Portal → **Microsoft Entra ID** → **アプリの登録** → **新規登録**
2. 入力:
   - **名前**: `SaaS管理ツール`
   - **サポートされるアカウントの種類**: 「この組織ディレクトリのみ（シングルテナント）」
   - **リダイレクトURI**: `Web` → `https://<APP_FQDN>/auth/entra_id/callback`
3. 「概要」ページから以下をメモ:
   - **アプリケーション (クライアント) ID** → `ENTRA_CLIENT_ID`
   - **ディレクトリ (テナント) ID** → `ENTRA_TENANT_ID`

### 7-2. クライアントシークレット作成

1. 「証明書とシークレット」→「新しいクライアントシークレット」
2. 説明: `SaaS管理ツール`、有効期限: 24ヶ月（推奨）
3. 作成された **値** をコピー → `ENTRA_CLIENT_SECRET`

> **この画面を離れると二度と確認できない。** 必ずコピーすること。

### 7-3. API アクセス許可の設定

「API のアクセス許可」→「アクセス許可の追加」→「Microsoft Graph」

**アプリケーションの許可:**

| 権限 | 用途 |
|------|------|
| `User.Read.All` | ユーザー一覧取得・退職者検出 |
| `GroupMember.Read.All` | グループメンバー同期 |
| `Application.Read.All` | エンタープライズアプリ（SaaS）一覧取得 |
| `Mail.Send` | Graph API 経由のメール送信 |

**委任されたアクセス許可:**

| 権限 | 用途 |
|------|------|
| `User.Read` | SSOログイン時のプロフィール取得 |

設定後、**「（テナント名）に管理者の同意を与えます」** をクリック。

### 7-4. Container Apps に Entra ID 情報を設定

**CLI:**

```bash
az containerapp update \
  --name app-dxceco-poc \
  --resource-group rg-dxceco-poc \
  --set-env-vars \
    "ENTRA_CLIENT_ID=<クライアントID>" \
    "ENTRA_CLIENT_SECRET=<シークレット値>" \
    "ENTRA_TENANT_ID=<テナントID>"
```

**GUI:** ステップ6と同様に **コンテナー** → **編集とデプロイ** → 環境変数に追加。

### 7-5. アクセス制御（任意）

特定ユーザー/グループだけにアクセスを制限する場合:

1. **Entra ID** → **エンタープライズ アプリケーション** → 対象アプリ
2. **プロパティ** → 「ユーザーの割り当てが必要ですか？」→ **はい**
3. **ユーザーとグループ** → アクセスを許可するユーザー/グループを追加

## 10. GitHub Actions CI/CD 設定

> このステップは GitHub の GUI 操作。

### 10-1. GitHub Secrets の登録

リポジトリの **Settings → Secrets and variables → Actions → New repository secret** で以下を登録:

| Secret 名 | 値 | 取得元 |
|-----------|-----|--------|
| `AZURE_CLIENT_ID` | サービスプリンシパルの appId | ステップ8 |
| `AZURE_CLIENT_SECRET` | サービスプリンシパルの password | ステップ8 |

## 12. CRONスケーリング設定

毎月1日 6:00 (JST) にコンテナを起動し、月次バッチを実行する。

### CLI

```bash
az containerapp update \
  --name app-dxceco-poc \
  --resource-group rg-dxceco-poc \
  --min-replicas 0 \
  --max-replicas 3 \
  --scale-rule-name monthly-batch \
  --scale-rule-type cron \
  --scale-rule-metadata \
    "timezone=Asia/Tokyo" \
    "start=0 6 1 * *" \
    "end=0 8 1 * *" \
    "desiredReplicas=1"
```

### GUI

1. Azure Portal → **コンテナー アプリ** → `app-dxceco-poc`
2. 左メニュー **アプリケーション** → **スケーリング**
3. **レプリカの範囲**: 最小 `0`、最大 `3`
4. **スケール ルールの追加**:
   - **ルール名**: `monthly-batch`
   - **種類**: `カスタム`
   - **カスタム ルールの種類**: `cron`
   - **メタデータ**:
     - `timezone` = `Asia/Tokyo`
     - `start` = `0 6 1 * *`
     - `end` = `0 8 1 * *`
     - `desiredReplicas` = `1`
5. **保存**

> 毎月1日 6:00〜8:00 (JST) に1レプリカを起動。8:00以降はアクセスがなければ自動停止（min 0）。

### Solid Queue の定期ジョブ

`config/recurring.yml` で月次バッチが定義済み。コンテナ起動時に Solid Queue が自動的にジョブを実行する。

## トラブルシューティング

### コンテナが起動しない

```bash
# ログ確認
az containerapp logs show --name app-dxceco-poc --resource-group rg-dxceco-poc --follow

# リビジョン状態確認
az containerapp revision list --name app-dxceco-poc --resource-group rg-dxceco-poc -o table
```

**GUI:** コンテナー アプリ → **監視** → **ログ ストリーム** / **リビジョン管理** で確認。

よくある原因:
- `RAILS_MASTER_KEY` が未設定 or 不正 → credentials が復号できない
- `DATABASE_URL` が不正 → DB接続失敗
- ポート設定の不一致 → ingress の `target-port` が 3000 であることを確認

### DB に接続できない

```bash
az postgres flexible-server firewall-rule list \
  --resource-group rg-dxceco-poc --name psql-dxceco-poc -o table
```

**GUI:** PostgreSQL → **設定** → **ネットワーク** でファイアウォール規則を確認。
「Azure 内の任意の Azure サービスにパブリック アクセスを許可する」にチェックが入っているか確認。

### SSO ログインに失敗する

- リダイレクトURI が `https://<FQDN>/auth/entra_id/callback` で正確に登録されているか
- `APP_URL` がアプリの実際のURLと一致しているか
- クライアントシークレットの有効期限が切れていないか
